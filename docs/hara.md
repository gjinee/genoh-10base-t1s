## 1. 시스템 설명 및 경계 (System Description and Boundaries)

### 1.1 아이템 정의 (Item Definition)

본 HARA의 분석 대상 아이템은 **SAM E70 MCU 기반 슬레이브 노드**로, 자동차 존(Zone) 아키텍처 내에서 다음 기능을 수행한다:

| 기능 | 설명 | 유형 |
|------|------|------|
| 헤드라이트 제어 | 마스터 명령에 따른 헤드라이트 LED ON/OFF 제어 | 액추에이터 |
| 비상 경고등(Hazard Blinker) 제어 | 비상등 점멸 패턴 제어 (1Hz 또는 2Hz) | 액추에이터 |
| 조향 센서 데이터 퍼블리시 | 조향각(steering angle) 센서 값을 주기적으로 마스터에 전송 | 센서 |
| Zenoh-pico 통신 | 10BASE-T1S 물리 버스 위 Zenoh 프로토콜 기반 Pub/Sub/Query | 통신 |
| E2E Protection | CRC-32, 시퀀스 카운터, 타임아웃 기반 통신 무결성 보호 | 안전 메커니즘 |
| 워치독 (Watchdog) | MCU 내부 WDT를 통한 소프트웨어 실행 감시 | 안전 메커니즘 |

## 2. 위험원 식별 (Hazard Identification)

### 2.1 ASIL 결정 기준 (ISO 26262 Part 3, Clause 7)

**심각도 (Severity, S)**

| 등급 | 설명 | 예시 |
|------|------|------|
| S0 | 부상 없음 | 편의 기능 장애 |
| S1 | 경상 및 중등도 부상 | 불편, 경미한 사고 유발 가능 |
| S2 | 중상(생존 가능, 치료 필요) | 심각한 사고 유발 가능 |
| S3 | 치명적 부상 또는 사망 | 생명 위협적 사고 |

**노출 확률 (Exposure, E)**

| 등급 | 설명 | 확률 |
|------|------|------|
| E0 | 상상할 수 없을 정도로 드묾 | — |
| E1 | 매우 낮은 확률 | <1% 운행 시간 |
| E2 | 낮은 확률 | 1~10% 운행 시간 |
| E3 | 중간 확률 | 10~50% 운행 시간 |
| E4 | 높은 확률 | >50% 운행 시간 |

**제어 가능성 (Controllability, C)**

| 등급 | 설명 | 운전자 대응 |
|------|------|-----------|
| C0 | 일반적으로 제어 가능 | 거의 모든 운전자 대응 가능 |
| C1 | 쉽게 제어 가능 | 99% 이상 운전자 대응 가능 |
| C2 | 대부분 제어 가능 | 90% 이상 운전자 대응 가능 |
| C3 | 제어 어려움 또는 불가능 | 90% 미만 운전자 대응 가능 |

### 2.3 위험원 분석 근거 상세

#### HAZ-001: 헤드라이트 주간 점등 고착
- **원인**: MCU GPIO 출력 High 고착, FET 드라이버 단락, Zenoh OFF 명령 수신 실패
- **심각도 S1 근거**: 주간이므로 직접적 충돌 위험 낮으나 대향차 운전자 눈부심으로 경미한 사고 가능, 장기적으로 배터리 방전
- **노출 E4 근거**: 주간 운행은 전체 운행 시간의 50% 이상
- **제어 C3 근거**: 운전자가 헤드라이트 고착을 즉시 인지하기 어려움 (대시보드 경고 의존)

#### HAZ-002: 헤드라이트 야간 소등 고착
- **원인**: MCU GPIO 출력 Low 고착, FET 드라이버 개방, Zenoh ON 명령 수신 실패, MCU 정지
- **심각도 S3 근거**: 야간 시인성 완전 상실은 정면 충돌/보행자 사고 직결, 치명적 부상 또는 사망 가능
- **노출 E3 근거**: 야간 운행은 전체 운행 시간의 약 30~40%
- **제어 C2 근거**: 운전자가 즉시 인지하고 감속/정차 가능하나, 고속도로에서는 대응 시간 부족

#### HAZ-003: 비상 경고등 점멸 실패
- **원인**: 점멸 타이머 오류, GPIO 제어 실패, Zenoh 비상등 명령 미수신
- **심각도 S2 근거**: 비상 상황에서 후속 차량 경고 불가로 추돌 위험, 중상 가능
- **노출 E3 근거**: 비상 상황 자체는 드물지만 비상등은 차선 변경/견인 등에도 사용, 누적 노출 중간 수준
- **제어 C2 근거**: 운전자가 수동 비상등 스위치로 대체 가능하나, 자동 비상등(충돌 감지 시)은 대체 불가

#### HAZ-004: 조향 센서 데이터 소실
- **원인**: Zenoh 세션 단절, UDP 패킷 손실, 슬레이브 타이머 중단, PHY 링크 다운
- **심각도 S3 근거**: ADAS/ESC가 조향 정보 없이 오판하면 차량 제어 상실, 치명적
- **노출 E4 근거**: 조향 데이터는 주행 중 상시 필요 (50% 이상 운행 시간)
- **제어 C2 근거**: 숙련된 운전자는 ADAS 경고로 수동 전환 가능하나, 고속 상황에서 한계

#### HAZ-005: 조향 센서 오류 데이터 전송
- **원인**: ADC 오프셋, EMI 간섭에 의한 데이터 비트 플립, 센서 캘리브레이션 오류
- **심각도 S3 근거**: 잘못된 조향각이 ADAS에 적용되면 의도치 않은 방향 전환, 치명적
- **노출 E4 근거**: 센서 데이터는 상시 전송
- **제어 C1 근거**: 오류 데이터에 의한 ADAS 오작동은 운전자가 감지 시 핸들 오버라이드로 대응 가능

#### HAZ-006: 의도하지 않은 액추에이터 활성화
- **원인**: MCU SRAM 비트 플립, 스택 오버플로우, Zenoh 메시지 오해석, 초기화 오류
- **심각도 S2 근거**: 임의 점등/점멸로 후속 차량 오인 가능, 전기 과부하 시 화재 위험
- **노출 E3 근거**: 전 운행 조건에서 발생 가능하나 정차 중 영향 제한적
- **제어 C3 근거**: 운전자가 즉시 인지하지 못할 수 있음 (뒤쪽 비상등 등)

#### HAZ-007: 통신 전면 두절
- **원인**: 10BASE-T1S 버스 물리적 단선, LAN8670 PHY 손상, 전원 손실, PLCA 비콘 소실
- **심각도 S3 근거**: 마스터-슬레이브 간 모든 제어/감시 불가, 안전 기능 전면 마비
- **노출 E3 근거**: 커넥터 진동, 환경 조건에 따라 중간 빈도 발생 가능
- **제어 C2 근거**: 계기판 통신 오류 경고로 운전자 인지 가능, 안전 정차 유도 가능

#### HAZ-008: 워치독 장애
- **원인**: WDT 레지스터 설정 오류, WDT 클록 소스 장애, 소프트웨어가 정상으로 위장(WDT 킥만 수행)
- **심각도 S2 근거**: 워치독 실패 자체보다 후속 장애(SW 정지 → 출력 고착)가 위험
- **노출 E2 근거**: WDT 하드웨어 자체 장애 확률 낮음, SW 미스 킥은 중간 빈도
- **제어 C3 근거**: 내부 메커니즘 장애는 운전자 직접 제어 불가

#### HAZ-009: CRC/E2E 오류 미감지
- **원인**: CRC 다항식 선택 오류, E2E 카운터 동기화 실패, 잔류 오류율(HDD) 초과
- **심각도 S3 근거**: 감지 우회된 손상 데이터가 제어에 직접 적용, 치명적 오작동 가능
- **노출 E3 근거**: EMI 환경에서 비트 오류 발생 빈도 중간, 10BASE-T1S는 CSMA 환경에서 충돌 가능
- **제어 C1 근거**: 오류가 미감지된 경우 시스템 레벨 플라우시빌리티 체크로 일부 검출 가능

#### HAZ-010: 보안 침해에 의한 미인가 명령
- **원인**: 10BASE-T1S 버스 물리적 접근(탭핑), MAC 위조, Zenoh 세션 하이재킹
- **심각도 S3 근거**: 임의 명령으로 헤드라이트 소등, 비상등 비활성화 등 치명적 상황 유발 가능
- **노출 E2 근거**: 물리적 버스 접근 필요, 일반 운행 환경에서 공격 기회 제한적
- **제어 C2 근거**: 운전자가 이상 작동 감지 시 수동 전환 가능하나, 즉시 인지 어려울 수 있음

### 3.3 ASIL 분해 가능성 (ASIL Decomposition)

ISO 26262 Part 9, Clause 5에 따라 독립적 요소에 대한 ASIL 분해가 가능하다:

| 원래 ASIL | 분해 조합 | 적용 조건 |
|-----------|----------|----------|
| ASIL-D | ASIL-D(D) + QM 또는 ASIL-C(D) + ASIL-A(D) 또는 ASIL-B(D) + ASIL-B(D) | 충분한 독립성 입증 필요 |
| ASIL-C | ASIL-C(C) + QM 또는 ASIL-B(C) + ASIL-A(C) | 충분한 독립성 입증 필요 |
| ASIL-B | ASIL-B(B) + QM 또는 ASIL-A(B) + ASIL-A(B) | 충분한 독립성 입증 필요 |

> **본 프로젝트 적용**: HAZ-002, HAZ-004 등 ASIL-D 항목에 대해, 슬레이브(SAM E70)와 마스터(RPi 5)의 독립적 감시(E2E + 워치독)를 통해 ASIL-B(D) + ASIL-B(D)로 분해하는 것을 목표로 한다.

## 4. 안전 목표 (Safety Goals)

ISO 26262 Part 3, Clause 8에 따라 위험원으로부터 안전 목표를 도출한다.

### 4.1 안전 목표 테이블

| ID | 안전 목표 (Safety Goal) | 관련 위험원 | ASIL | 안전 상태 (Safe State) | FTTI |
|----|----------------------|-----------|------|---------------------|------|
| SG-001 | 헤드라이트는 야간 운행 시 반드시 점등 상태를 유지해야 하며, 주간에는 명령에 따라 소등할 수 있어야 한다 | HAZ-001, HAZ-002 | ASIL-D | 야간: 헤드라이트 강제 점등 (Fail-ON), 주간: 소등 허용 | 100ms |
| SG-002 | 비상 경고등은 비상 명령 수신 시 반드시 정상 점멸해야 한다 | HAZ-003 | ASIL-C | 비상등 강제 점멸 (독립 타이머) | 200ms |
| SG-003 | 조향 센서 데이터는 정확하고 완전하게 마스터에 전달되어야 하며, 이상 시 데이터 유효성이 표시되어야 한다 | HAZ-004, HAZ-005 | ASIL-D | 센서 데이터에 유효성 플래그 = INVALID 설정, 마스터는 최종 유효 값 또는 기본값 사용 | 50ms |
| SG-004 | 통신 단절 또는 워치독 장애 시 시스템은 안전 상태로 전이해야 한다 | HAZ-006, HAZ-007, HAZ-008 | ASIL-D | 독립 안전 상태: 헤드라이트 ON, 비상등 점멸, 액추에이터 동결(Freeze) | 100ms |
| SG-005 | 통신 데이터 무결성이 보장되어야 하며, 미인가 명령이 실행되지 않아야 한다 | HAZ-009, HAZ-010 | ASIL-D | E2E 오류 감지 시 해당 메시지 폐기, 3회 연속 오류 시 안전 상태 전이 | 50ms |

### 4.2 안전 목표 상세 설명

#### SG-001: 헤드라이트 안전 점등 보장
- **근거**: HAZ-002(ASIL-D)가 가장 높은 위험을 나타내므로, 야간 헤드라이트 소등 방지가 최우선
- **Fail-Safe 전략**: 야간 모드에서 통신 두절 시 헤드라이트 ON 유지 (Fail-ON), GPIO 기본 상태를 HIGH로 설정
- **FTTI(Fault Tolerant Time Interval)**: 100ms — 차량 속도 100km/h에서 약 2.8m 이동 거리, 야간 시인성 확보를 위해 100ms 이내 장애 감지 필요

#### SG-002: 비상 경고등 신뢰성 보장
- **근거**: HAZ-003(ASIL-C)은 비상 상황에서의 2차 사고 방지에 핵심
- **독립 타이머**: Zenoh 통신과 무관하게 MCU 내부 타이머로 점멸 패턴 유지
- **FTTI**: 200ms — 비상등 점멸 주기(500ms)의 절반 이내에 장애 감지

#### SG-003: 조향 센서 데이터 무결성
- **근거**: HAZ-004, HAZ-005 모두 ASIL-D로 조향 데이터의 정확성과 적시성이 필수
- **유효성 관리**: 각 센서 메시지에 validity 플래그, timestamp, 시퀀스 카운터 포함
- **FTTI**: 50ms — ADAS 제어 주기(20~50ms)에 부합, 데이터 소실 1주기 이내 감지

#### SG-004: 통신 두절/워치독 장애 시 안전 상태 전이
- **근거**: HAZ-007(ASIL-D)은 전면적 기능 마비를 의미, 독립적 안전 상태 필요
- **안전 상태 머신**: NORMAL → DEGRADED → SAFE 3단계 전이
  - DEGRADED: 통신 간헐 장애, 최종 유효 명령 유지
  - SAFE: 통신 완전 두절(>500ms), 헤드라이트 ON + 비상등 점멸
- **FTTI**: 100ms — 워치독 타임아웃을 100ms로 설정

#### SG-005: 데이터 무결성 및 인증
- **근거**: HAZ-009(ASIL-D), HAZ-010(ASIL-C)으로 무결성과 인증 모두 필요
- **E2E Protection**: AUTOSAR E2E Profile 4 호환 — CRC-32, 시퀀스 카운터, 타임아웃
- **SecOC**: HMAC-SHA256 기반 메시지 인증, 리플레이 방지
- **FTTI**: 50ms — 오류 감지 후 즉시 메시지 폐기, 연속 오류 시 안전 상태 전이
